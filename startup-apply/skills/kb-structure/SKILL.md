# 지식베이스 구조

startup-apply 플러그인의 모든 사업계획서 자동화는 이 지식베이스를 기반으로 동작합니다. 한 번 구축하면 모든 공고의 사업계획서를 반복 없이 재활용합니다.

## 7개 카테고리 스키마

### 1. company-profile (기업 기본 정보)

**필수:**
- 법인명 (+ 영문명)
- 설립일
- 대표자 성명
- 소재지 (주소)
- 사업자등록번호
- 업종 코드 (KSIC)
- 기업 형태 (주식회사, 유한회사 등)
- 미션 / 비전 (한 문장)

**선택:**
- 웹사이트 / SNS
- 연혁 (연도별 주요 사건)
- 수상 / 인증 이력

**팀 구성 (핵심 인력별):**
- 이름 / 직책
- 최종 학력 (대학, 전공)
- 주요 경력 (3줄 이내)
- 담당 역할

---

### 2. product (제품 / 서비스)

**필수:**
- 제품/서비스 한 줄 설명
- 해결하는 문제
- 핵심 기능 목록
- 기술 차별점 (경쟁 대비)

**선택:**
- 기술 스택
- 개발 단계 (아이디어 / MVP / 베타 / 정식 출시)
- 서비스 URL / 데모

**기술 / IP:**
- 특허 (출원 / 등록, 번호, 명칭)
- 소프트웨어 저작권
- 인증 (ISO, KC, GS 등)
- TRL 단계 (1~9)

---

### 3. market (시장 분석)

**필수:**
- TAM (전체 시장 규모, 출처, 산출 근거)
- SAM (서비스 가능 시장, 출처, 산출 근거)
- SOM (실제 목표 시장, 산출 근거)
- 목표 고객 (페르소나 1~3개)

**선택:**
- 시장 성장률 (CAGR, 출처)
- 경쟁사 분석 (이름, 강점, 약점, 우리와의 차별점)
- 시장 트렌드 (3~5개 키워드)

> 주의: TAM/SAM/SOM은 반드시 출처(보고서명, 발행기관, 연도)와 산출 근거를 함께 기재해야 평가에서 인정받습니다.

---

### 4. business-model (비즈니스 모델)

**필수:**
- 수익 모델 (구독/거래/라이선스/광고 등)
- 주요 고객 세그먼트
- 가격 정책 (플랜별 가격)
- 고객 획득 채널

**선택:**
- GTM 전략 요약
- 핵심 파트너사
- 주요 고객/레퍼런스

---

### 5. financials (재무 / 지표)

**필수:**
- 매출 추이 (연도별, 단위: 만원)
- 현재 MRR / ARR
- 손익분기점 (BEP) 달성 여부 / 예상 시점

**선택:**
- 주요 KPI:
  - MAU / DAU
  - 전환율 (CVR)
  - LTV / CAC 비율
  - NPS
- 재무제표 요약 (매출, 영업이익, 당기순이익)
- 투자 유치 이력 (라운드, 금액, 투자자)
- 현재 런웨이 (개월)
- 자금 사용 계획 (항목별 %)

---

### 6. track-record (수행 실적)

**필수:**
- 과거 지원사업 수행 이력:
  - 사업명, 주관기관, 연도, 지원금액, 성과
- 수상 / 선정 이력

**선택:**
- 투자 유치 이력
- 주요 고객사 / 파트너사 로고
- 언론 보도

---

### 7. past-plans (과거 문서 패턴)

자동 생성됨 — `/kb-init` 실행 시 과거 사업계획서를 분석하여 채워집니다.

- 자주 쓰는 문체 / 톤 특성
- 섹션별 표현 패턴 (예: "~을 통해 ~를 실현합니다")
- 강조하는 차별점 키워드
- 공고 유형별 작성 패턴

## 완성도 점수 기준

| 카테고리 | 가중치 | 기준 |
|----------|-------|------|
| company-profile | 15% | 필수 항목 전체 + 팀 2인 이상 |
| product | 20% | 필수 항목 전체 + 기술/IP 1건 이상 |
| market | 20% | TAM/SAM/SOM 출처 포함 |
| business-model | 15% | 수익 모델 + 가격 정책 |
| financials | 20% | 매출 추이 + KPI 3개 이상 |
| track-record | 10% | 실적 1건 이상 |

**완성도 해석:**
- 90%+ → 모든 지원사업 즉시 지원 가능
- 70~89% → 대부분 지원 가능, 일부 보완 필요
- 50~69% → 주요 공고 지원 전 보완 필요
- 50% 미만 → `/kb-update`로 핵심 항목 채우기 선행

## 카테고리별 작성 가이드

**market — TAM/SAM/SOM 작성 원칙**
- 반드시 출처 명기: "○○연구원, 2025년 국내 AI 시장 전망 보고서"
- 산출 근거 제시: "국내 중소기업 수 80만 × 평균 SaaS 지출 연 200만원 = TAM 1.6조원"
- 가장 보수적인 숫자를 사용할 것

**financials — 수치 제시 원칙**
- 예정이 아닌 검증된 수치 우선: "2025년 4월 MRR 1,200만원 달성"
- 성장률 제시 시 기간 명확히: "전년 동기 대비 MoM 15% 성장 (2025.01~06)"

**product — 기술 차별점 작성 원칙**
- 경쟁사 대비 정량적 차이 제시: "처리 속도 3배, 비용 40% 절감"
- TRL 단계 명확히: "TRL 7 (시제품 검증 완료)"
