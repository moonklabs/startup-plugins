# /apply-write

> 익숙하지 않은 플레이스홀더가 보이거나 연결된 도구를 확인하려면 [CONNECTORS.md](../CONNECTORS.md)를 참조하세요.

지식베이스를 기반으로 지원사업 사업계획서를 자동 작성합니다. 공고를 분석하고, 필요한 데이터를 KB에서 로드하며, 섹션별로 초안을 작성합니다.

## 사용법

```
/apply-write                           # 대화형으로 공고 선택
/apply-write TIPS 2026년 상반기        # 공고 지정
/apply-write 창업성장기술개발           # 공고명만 지정
/apply-write TIPS --continue           # 이어서 작성
```

---

## 작동 방식

```
┌─────────────────────────────────────────────────────────────────┐
│                    APPLY WRITE                                   │
├─────────────────────────────────────────────────────────────────┤
│  단독 사용 (항상 작동)                                            │
│  ✓ 공고명으로 평가 기준/배점/양식 자동 검색                        │
│  ✓ 지식베이스에서 필요한 회사 정보 자동 로드                       │
│  ✓ bizplan-writer 에이전트가 섹션별 초안 작성                     │
│  ✓ 배점 높은 섹션부터 우선 작성                                   │
│  ✓ 최종 Markdown 파일로 저장                                      │
├─────────────────────────────────────────────────────────────────┤
│  강화 모드 (도구 연결 시)                                         │
│  + ~~knowledge base: Notion KB 자동 참조 (최신 데이터)            │
│  + ~~data enrichment: 시장 데이터 실시간 보강                    │
│  + ~~docs: 과거 유사 사업계획서 자동 참조                         │
└─────────────────────────────────────────────────────────────────┘
```

## 4단계 워크플로우

```
/apply-write TIPS 2026년 상반기
        │
        ▼
┌───────────────────────────────────────────┐
│ 1단계: 공고 분석                           │
│  웹 검색 → 공고 요강 파악                  │
│  - 평가 항목 및 배점                       │
│  - 페이지/글자수 제한                      │
│  - 제출 양식 유무                          │
│  - 필수 기재 항목 체크리스트                │
└───────────────────────────────────────────┘
        │
        ▼
┌───────────────────────────────────────────┐
│ 2단계: 지식베이스 매핑                     │
│  KB ↔ 공고 필수 항목 대조                  │
│  - 충분: 바로 작성 진행                    │
│  - 부족: 사용자에게 알림                   │
│    → "시장 규모 출처가 없습니다.            │
│       지금 추가하거나 작성 후 보완?"        │
└───────────────────────────────────────────┘
        │
        ▼
┌───────────────────────────────────────────┐
│ 3단계: 섹션별 작성                         │
│  bizplan-writer 에이전트                   │
│  (배점 높은 순서로, 독립 섹션은 병렬)       │
│                                           │
│  각 섹션 완료 후 사용자 검토 옵션:          │
│  "기술 개발 내용 초안입니다. 계속할까요?"   │
└───────────────────────────────────────────┘
        │
        ▼
┌───────────────────────────────────────────┐
│ 4단계: 최종 조립                           │
│  - 전체 문서 일관성 검토                   │
│    (용어 통일, 수치 교차검증)              │
│  - 페이지/글자수 제한 준수 확인            │
│  - Markdown 최종본 저장                    │
│  → /apply-export로 HWPX 변환 가능         │
└───────────────────────────────────────────┘
```

---

```markdown
## 1. 기업 개요
[내용]

## 2. 대표자 및 팀 역량
[내용]

...

## 13. 수행 실적
[내용]

---

## 작성 완료 후 다음 단계
→ HWPX 파일 생성: `/apply-export [공고명]`
→ 다른 공고에 재활용: `/apply-update [파일명] → [새 공고명]`
```

## 관련 커맨드

- `/kb-update` — 사업계획서 작성 전 KB 최신화
- `/apply-check` — 지원 전 적합도 확인
- `/apply-update` — 기존 사업계획서 재활용
- `/apply-export` — HWPX 파일로 내보내기

## 관련 스킬

이 커맨드 실행 시 다음 스킬이 자동 활성화됩니다:
- `bizplan-writing` — 공고 유형별 작성 전략
- `kb-structure` — KB 스키마 참조
