# /kb-init

> 익숙하지 않은 플레이스홀더가 보이거나 연결된 도구를 확인하려면 [CONNECTORS.md](../CONNECTORS.md)를 참조하세요.

회사의 모든 과거 문서를 한 번에 정리하여 지식베이스를 구축합니다. 한 번 실행하면 이후 모든 사업계획서 작성의 기반이 됩니다.

## 사용법

```
/kb-init
```

---

## 작동 방식

```
┌─────────────────────────────────────────────────────────────────┐
│                      KB INIT                                     │
├─────────────────────────────────────────────────────────────────┤
│  단독 사용 (항상 작동)                                            │
│  ✓ 과거 사업계획서 / IR자료 / 회사소개서 파일 업로드               │
│  ✓ kb-extractor 에이전트가 자동으로 정보 추출 + 분류              │
│  ✓ 7개 카테고리 스키마에 매핑                                      │
│  ✓ 빈 항목 식별 → 보완 가이드 제공                                 │
│  ✓ 로컬 .kb/ 폴더에 Markdown으로 저장                             │
├─────────────────────────────────────────────────────────────────┤
│  강화 모드 (도구 연결 시)                                         │
│  + ~~knowledge base: Notion에 자동으로 KB 페이지 생성              │
│  + ~~docs: 연결된 문서 도구에서 기존 파일 자동 탐색                │
└─────────────────────────────────────────────────────────────────┘
```

## 워크플로우

### 1단계: 문서 수집

**~~docs 미연결 시:**
> "과거 사업계획서, IR 자료, 회사 소개서 파일을 올려주세요. (Word, PDF, TXT 형식 권장 — HWP는 텍스트 추출이 제한될 수 있습니다)"

**~~docs 연결 시:**
연결된 문서 도구에서 "사업계획서", "IR", "회사소개" 키워드로 자동 탐색하여 후보 파일 목록을 제시합니다.

### 2단계: 자동 추출 (kb-extractor 에이전트)

```
문서 A ──┐
문서 B ──┼── kb-extractor 에이전트 (병렬) ──→ 카테고리별 정보 추출
문서 C ──┘
```

각 문서에서 7개 카테고리(company-profile, product, market, business-model, financials, track-record, past-plans)로 자동 분류합니다.

### 3단계: 검토 및 보완

추출 결과를 요약하고 사용자에게 확인 요청:

```
✅ company-profile: 85% 완성 (법인명, 대표자, 팀 구성 확인됨)
✅ product: 90% 완성 (기술 스택, 특허 2건 확인됨)
⚠️  market: 60% 완성 → TAM/SAM 있으나 SOM 및 출처 없음
⚠️  financials: 50% 완성 → 최신 MRR 데이터 없음
✅ track-record: 95% 완성
```

빈 항목은 즉시 채우거나 나중에 `/kb-update`로 보완할 수 있습니다.

### 4단계: 저장

**~~knowledge base(Notion) 연결 시:**
Notion에 "[회사명] 지식베이스" 페이지를 생성하고 카테고리별 하위 페이지를 자동 작성합니다.

**미연결 시:**
프로젝트 루트의 `.kb/` 폴더에 카테고리별 Markdown 파일로 저장합니다.

## 출력 예시

```markdown
# 지식베이스 구축 완료 | 2026-02-23

## 완성도 요약

| 카테고리 | 완성도 | 상태 |
|----------|-------|------|
| company-profile | 85% | ✅ |
| product | 90% | ✅ |
| market | 60% | ⚠️ 보완 필요 |
| business-model | 80% | ✅ |
| financials | 50% | ⚠️ 보완 필요 |
| track-record | 95% | ✅ |
| past-plans | 자동 생성 | ✅ |

**종합 완성도: 77%**

## 보완 권고 항목

1. `market.SOM` — 산출 근거 없음
2. `market.TAM 출처` — 보고서 출처 명기 필요
3. `financials.MRR` — 최신 수치 입력 필요

→ 지금 보완: `/kb-update market`, `/kb-update financials`
→ 나중에 보완: 지원사업 작성 전 `/kb-update --check`로 점검

## 저장 위치
- Notion: [회사명] 지식베이스 / (또는) .kb/ 폴더
```

## 관련 커맨드

- `/kb-update` — 지식베이스 갱신
- `/apply-write` — 사업계획서 작성 시작
- `/apply-daily` — 전체 지원사업 현황 파악

## 관련 스킬

이 커맨드 실행 시 `kb-structure` 스킬이 자동 활성화됩니다.
