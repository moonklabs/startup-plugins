# /apply-daily

> 익숙하지 않은 플레이스홀더가 보이거나 연결된 도구를 확인하려면 [CONNECTORS.md](../CONNECTORS.md)를 참조하세요.

오늘의 지원사업 현황을 한눈에 파악합니다. 마감 임박 공고, 작성 진행률, 신규 공고 알림, 지식베이스 상태를 2분 안에 스캔할 수 있는 브리핑을 제공합니다.

## 사용법

```
/apply-daily
```

---

## 작동 방식

```
┌─────────────────────────────────────────────────────────────────┐
│                  APPLY DAILY BRIEFING                            │
├─────────────────────────────────────────────────────────────────┤
│  단독 사용 (항상 작동)                                            │
│  ✓ 사용자 제공: 현재 진행 중인 지원사업 현황                       │
│  ✓ 마감 임박 공고 우선순위 정리                                    │
│  ✓ 각 사업계획서 작성 진행률 체크                                  │
│  ✓ 오늘의 체크리스트 생성                                         │
├─────────────────────────────────────────────────────────────────┤
│  강화 모드 (도구 연결 시)                                         │
│  + ~~knowledge base: 파이프라인 현황 자동 로드 (Notion DB)         │
│  + ~~data enrichment: 신규 공고 자동 탐지 (웹 검색)               │
│  + ~~chat: 데일리 리포트 팀 채널에 자동 공유                       │
└─────────────────────────────────────────────────────────────────┘
```

## 필요한 정보

**~~knowledge base 미연결 시:**
> "현재 진행 중인 지원사업을 알려주세요. (공고명, 마감일, 작성 진행률)"

**~~knowledge base 연결 시:**
Notion의 지원사업 파이프라인 데이터베이스에서 자동으로 로드합니다.

---

## 파이프라인 추적 구조

Notion DB 또는 로컬 파일로 다음 항목을 관리합니다:

| 항목 | 설명 |
|------|------|
| 공고명 | 지원사업명 |
| 주관기관 | 주관 기관 |
| 마감일 | 제출 마감일 |
| 상태 | 소싱/분석/작성중/검토/제출/결과대기/선정/탈락 |
| 적합도 | % 점수 |
| 작성률 | % (완성된 섹션 비율) |
| 파일 경로 | 사업계획서 Markdown 파일 위치 |
| 메모 | 특이사항 |


## 우선순위 프레임워크

| 항목 | 가중치 | 기준 |
|------|-------|------|
| 마감 임박 | 35% | D-7 이내이면 최우선 |
| 작성률 낮음 | 30% | 마감까지 완성 가능한지 역산 |
| KB 미갱신 | 20% | 30일 이상 미갱신 항목 있는 경우 |
| 신규 공고 | 15% | 적합도 80%+ 신규 공고 발견 시 |

## 관련 커맨드

- `/apply-find` — 신규 공고 소싱
- `/apply-write` — 사업계획서 작성 시작 또는 이어쓰기
- `/apply-check` — 특정 공고 적합도 확인
- `/kb-update` — 지식베이스 갱신
