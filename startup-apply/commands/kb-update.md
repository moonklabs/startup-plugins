# /kb-update

> 익숙하지 않은 플레이스홀더가 보이거나 연결된 도구를 확인하려면 [CONNECTORS.md](../CONNECTORS.md)를 참조하세요.

지식베이스의 특정 카테고리를 업데이트하거나, 새 문서에서 정보를 추출하거나, 미갱신 항목을 점검합니다.

## 사용법

```
/kb-update                        # 대화형으로 업데이트 항목 선택
/kb-update financials             # 특정 카테고리 업데이트
/kb-update market                 # 시장 분석 데이터 갱신
/kb-update --from 파일.docx       # 새 문서에서 자동 추출
/kb-update --check                # 미갱신 항목 점검
```

## 모드별 워크플로우

### 모드 1: 카테고리 업데이트

```
/kb-update financials
```

현재 저장된 financials 데이터를 보여주고 항목별 업데이트를 진행합니다:

```
현재 financials 데이터 (마지막 갱신: 2025-11-01)

- 매출 (2025): 2.4억원 → 최신 값은?
- MRR (현재): 2,000만원 → 최신 값은?
- 주요 KPI:
  - MAU: 5,200 → 최신 값은?
  ...
```

### 모드 2: 문서 기반 추출

```
/kb-update --from 2026_1분기보고서.pdf
```

1. 문서 로드 → kb-extractor 에이전트로 정보 추출
2. 기존 KB와 비교하여 변경된 항목 식별
3. 변경 내역을 diff 형식으로 표시 후 승인 요청:

```
📄 2026_1분기보고서.pdf 분석 완료

변경 감지:
+ financials.MRR: 2,000만원 → 2,800만원 (변경)
+ financials.MAU: 5,200 → 7,100 (변경)
+ track-record: "2026 CES 혁신상 수상" (추가)
~ market.TAM: 변경 없음

변경 사항을 적용할까요? [y/n]
```

### 모드 3: 점검

```
/kb-update --check
```

```markdown
# 지식베이스 점검 결과 | 2026-02-23

## 미갱신 항목 (30일 이상)

| 카테고리 | 항목 | 마지막 갱신 | 우선순위 |
|----------|------|------------|--------|
| financials | MRR, 매출 추이 | 45일 전 | 🔴 높음 |
| market | TAM 출처 업데이트 | 90일 전 | 🟡 보통 |
| track-record | 최근 수상 이력 | 60일 전 | 🟡 보통 |

## 사업계획서 작성 전 필수 확인

다음 공고 작성 시 영향을 받는 항목:
- TIPS 2026 작성 시 → financials.MRR 최신화 필요
- 창업성장기술개발 작성 시 → market.SOM 출처 추가 필요

→ 즉시 갱신: `/kb-update financials`
```

## 출력 예시

갱신 완료 후:

```markdown
# 지식베이스 갱신 완료 | 2026-02-23

## 변경 사항
- financials.MRR: 2,000만원 → 2,800만원 ✅
- financials.MAU: 5,200 → 7,100 ✅
- track-record: "2026 CES 혁신상" 추가 ✅

## 현재 완성도
종합: 87% (+10%)

→ 사업계획서 작성 시작: `/apply-write [공고명]`
```

## 관련 커맨드

- `/kb-init` — 지식베이스 최초 구축
- `/apply-write` — 사업계획서 작성
- `/apply-daily` — 전체 현황 파악

## 관련 스킬

이 커맨드 실행 시 `kb-structure` 스킬이 자동 활성화됩니다.
