# /business-case

> 익숙하지 않은 플레이스홀더가 보이거나 연결된 도구를 확인하려면 [CONNECTORS.md](../CONNECTORS.md)를 참조하세요.

투자자, 전략적 파트너, 또는 이사회를 위한 종합 비즈니스 케이스 문서를 생성합니다. Executive Summary부터 투자 제안까지 10개 섹션으로 구성된 설득력 있는 비즈니스 스토리를 마크다운 파일로 저장합니다.

## 사용법

```bash
/business-case                    # 대화형으로 정보 수집
/business-case "MyStartup"        # 회사명 지정
/business-case "Series A 준비"    # 목적 지정
```

## 작동 방식

```
┌─────────────────────────────────────────────────────────────────┐
│                    BUSINESS CASE GENERATOR                       │
├─────────────────────────────────────────────────────────────────┤
│  기본 기능 (단독 작동)                                            │
│  ✓ 대화형 정보 수집 (회사, 시장, 팀, 트랙션)                      │
│  ✓ 10개 섹션 구조화 문서 자동 생성                                │
│  ✓ 관련 스킬 활성화 (market-sizing, financial-modeling 등)      │
│  ✓ 마크다운 파일 저장 (business-case-[회사명]-YYYY-MM-DD.md)   │
├─────────────────────────────────────────────────────────────────┤
│  강화 모드 (도구 연결 시)                                         │

│  + ~~data enrichment: 시장 데이터 자동 수집                      │
│  + ~~spreadsheet: 재무 모델 연동                                │
│  + ~~docs: 기존 문서에서 콘텐츠 재사용                           │
└─────────────────────────────────────────────────────────────────┘
```

## 필요한 정보

### 1단계: 기본 정보 수집

**회사 기본 정보**:
- 회사명
- 설립 연도
- 팀 규모
- 본사 위치
- 웹사이트/소셜

**비즈니스 모델**:
- 제품/서비스 한 줄 설명
- 타겟 고객 (ICP)
- 수익 모델 (SaaS/마켓플레이스/컨슈머/B2B)
- 가격 (ARPA 또는 ACV)

**목적**:
- 문서 대상 독자 (VC, 엔젤, 전략적 투자자, 이사회)
- 투자 라운드 (Pre-seed, Seed, Series A/B)
- 목표 금액

### 2단계: 기존 자료 확인

**~~knowledge base** 또는 **~~docs** 연결 시:
- 기존 피치 덱 참조
- 재무 모델 데이터 추출
- 팀 정보 자동 수집

연결 안 된 경우:
- 사용자에게 기존 자료 업로드 또는 붙여넣기 요청
- 없으면 대화형으로 정보 수집

### 3단계: 관련 스킬 자동 활성화

문서 생성 중 다음 스킬을 백그라운드에서 활용:

| 섹션 | 활성화 스킬 | 목적 |
|------|-------------|------|
| 시장 기회 | **market-sizing** | TAM/SAM/SOM 계산 |
| 재무 전망 | **financial-modeling** | 3-시나리오 매출/비용 |
| 경쟁 분석 | **competitive-landscape** | Porter's 5 Forces, 포지셔닝 |
| 핵심 지표 | **startup-metrics** | 비즈니스 모델별 KPI |

## 팁

- **간결함 우선**: 각 섹션 1-2페이지 목표. 상세는 부록으로
- **데이터 기반**: "우리 생각에"보다 "고객 인터뷰 30개 결과"
- **스토리텔링**: 숫자만 나열 말고 "왜" 중요한지 설명
- **시각화**: 복잡한 개념은 다이어그램으로 (Mermaid 활용)
- **일관성**: 같은 수치(ARR, 고객 수)가 섹션마다 다르면 안 됨
- **현실성**: 과도한 낙관 금지. 보수적 Base Case + Upside
- **대상 맞춤**: VC용 vs 이사회용은 강조점 다름
  - VC: 시장/성장/Exit
  - 이사회: 실행/리스크/재무 건전성
- **정기 업데이트**: 분기별 수정 (트랙션, 재무 실적 반영)
- **공동 작업**: 창업팀 전체가 리뷰, 일관된 메시지 확인
