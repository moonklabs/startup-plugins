# /investor-update

> 💡 **커넥터 참조:** 이 커맨드는 [`CONNECTORS.md`](../CONNECTORS.md)에 정의된 도구 플레이스홀더를 사용합니다. 연결된 도구가 많을수록 더 강력해지지만, 사용자 입력만으로도 효과적으로 작동합니다.

## 설명

기존 투자자 및 잠재 투자자를 위한 월간 또는 분기 업데이트를 작성합니다. 투명성과 신뢰 구축을 위해 성과와 챌린지를 균형 있게 공유하고, 구체적인 도움 요청을 포함하며, 일관된 형식으로 진행 상황을 추적합니다.

**핵심 가치:**
- 투명성: 좋은 소식과 어려운 소식 모두 솔직하게
- 일관성: 매달 같은 형식, 같은 날짜
- 구체성: 모호한 표현 대신 구체적 숫자와 날짜
- 실행 가능성: 막연한 "도움" 대신 구체적 ask

## 사용법

```bash
# 월간 업데이트
/investor-update "11월"

# 분기 업데이트
/investor-update "Q4 2024"

# 특정 대상 지정
/investor-update "11월 - 기존 투자자"

# 특별 업데이트
/investor-update "Series A 킥오프 공지"
```

## 업데이트 구조 (7섹션)

### 1. TL;DR (3줄)

```
목적: 바쁜 투자자가 30초 만에 핵심 파악

원칙:
- 가장 중요한 3가지만
- 구체적 숫자 포함
- 긍정/부정 균형

예시:
- MRR 5천만원 돌파 (+25% MoM)
- [대기업 A] POC 확정
- 시니어 엔지니어 2명 조인

피하기:
- "좋은 한 달이었습니다"
- "많은 진전이 있었습니다"
- 모호한 표현
```

### 2. 핵심 지표 표

```
선택 기준:
- 비즈니스 모델 핵심 지표 3-5개
- 투자자가 트래킹하는 지표
- 매달 일관된 지표 (추이 파악)

SaaS:
- MRR, ARR
- 고객 수
- Churn rate
- NRR (Net Revenue Retention)

마켓플레이스:
- GMV
- Take rate
- 활성 구매자/판매자
- 반복 거래율

E-commerce:
- GMV
- 주문 수
- AOV
- 재구매율

표 형식:
| 지표 | 이번 달 | 지난 달 | 변화 | 목표 대비 |
```

### 3. 하이라이트 (3-4개)

```
카테고리 (4가지 중 선택):

1. 제품
   - 신규 기능 출시
   - 성능 개선
   - 사용자 피드백

2. 고객
   - 주요 고객 확보
   - POC/파일럿 시작
   - NPS/만족도 개선

3. 팀
   - 주요 인력 채용
   - 조직 변경
   - 문화 개선

4. 자금
   - 런웨이 현황
   - Burn rate 변화
   - 투자 유치 진행

원칙:
- 각 하이라이트에 영향/의미 설명
- 구체적 숫자 포함
- "곧", "진행 중" 대신 날짜
```

### 4. 챌린지 & 학습

```
목적:
- 투명성 유지
- 리스크 조기 공유
- 대응 능력 증명

구조:
[챌린지] → [우리의 대응] → [현재 상태/학습]

예시:
"엔터프라이즈 세일즈 사이클 예상보다 김 (45일 → 60일)
→ POC 프로세스 표준화 중
→ 다음 달부터 적용, SMB 병행 공략"

원칙:
- 챌린지 숨기지 말기
- 하지만 해결 의지 명확히
- 학습 포인트 강조
```

### 5. 도움 요청

```
구체성이 핵심:

❌ 나쁜 예:
- "채용에 도움 주세요"
- "고객 소개 부탁드립니다"

✅ 좋은 예:
- "Head of Sales 후보 추천 (엔터프라이즈 경험 5년+)"
- "[대기업 B] IT 담당자 인트로 (POC 제안 중)"
- "엔터프라이즈 세일즈 프로세스 조언 (30분 콜)"

형식:
- [ ] 체크박스로 실행 가능하게
- 2-3개로 제한 (너무 많으면 무시됨)
- 구체적 요청 + 맥락
```

### 6. 다음 달 목표

```
SMART 목표:
- Specific: MRR 6.5천만원 (구체적)
- Measurable: 측정 가능
- Achievable: 달성 가능
- Relevant: 관련성
- Time-bound: by 12/31 (날짜)

개수:
- 3-4개로 제한
- 너무 많으면 집중력 분산

예시:
- MRR 6.5천만원 by 12/31
- [대기업 A] POC 킥오프
- CTO 영입 완료
- Activation 75%+ 달성
```

### 7. 감사 인사 & 다음 업데이트 예고

```
간단하지만 진심 어린:

"항상 지원해 주셔서 감사합니다.
자세한 내용은 언제든 논의 가능합니다.

다음 업데이트: 1월 5일"

포인트:
- 다음 업데이트 날짜 명시 (예측 가능성)
- 열린 커뮤니케이션 강조
```

## 발송 케이던스 & 타이밍

### 월간 업데이트

```
추천 일정:
- 매달 첫째 주 금요일 (일관성)
- 익월 5일 이내 발송

이유:
- 월말 데이터 집계 시간 확보
- 투자자 월요일 피크 타임 피함
- 일관된 날짜로 예측 가능성
```

### 분기 업데이트

```
형식:
- 월간보다 더 전략적
- 3개월 트렌드 분석
- 다음 분기 목표 상세

발송:
- 분기 종료 후 10일 이내
```

### Ad-hoc 업데이트

```
언제:
- 중요 마일스톤 (대형 고객 확보, 펀딩 클로징)
- 위기 상황 (24시간 이내)
- 방향 전환
- 주요 팀 변화

원칙:
- 좋은 소식: 즉시
- 나쁜 소식: 더 즉시
```

## 스타일 가이드

### Do's

✅ **구체적 숫자**
- "MRR 5천만원 (+25%)" ○
- "매출이 크게 성장" ×

✅ **날짜 명시**
- "12월 31일까지" ○
- "곧", "조만간" ×

✅ **투명한 챌린지**
- "세일즈 사이클 60일 (목표 45일)" ○
- "몇 가지 어려움" ×

✅ **구체적 도움 요청**
- "Head of Sales 후보 추천" ○
- "채용에 도움 주세요" ×

### Don'ts

❌ **마크다운 서식**
- 이메일에서 **볼드**, *이탤릭* 렌더링 안 됨
- 유니코드 구분선 사용 (━━━)

❌ **장황한 설명**
- TL;DR은 3줄
- 하이라이트는 각 2-3문장
- 전체 300-500단어

❌ **변명**
- "시장 상황이 어려워서..."
- "예상치 못한 문제로..."
- → 대신 대응 방안 강조

❌ **과장**
- "폭발적 성장"
- "게임 체인저"
- → 구체적 숫자로 대체

## 팁

1. **일관성이 신뢰를 만듭니다** — 매달 같은 날짜, 같은 형식으로 발송하세요.

2. **TL;DR은 정말 중요합니다** — 80%의 투자자는 TL;DR만 읽습니다.

3. **챌린지 공유는 강점입니다** — 투명성이 신뢰를 쌓습니다.

4. **데이터가 스토리를 만듭니다** — 숫자 없는 업데이트는 공허합니다.

5. **도움 요청은 구체적으로** — 막연한 요청은 무시됩니다.

6. **좋은 소식 먼저? NO** — 중요한 것 먼저. 나쁜 소식이 중요하면 먼저.

7. **지난 달 목표 추적** — 지난 달 "다음 달 목표"와 비교하여 accountability 유지.

8. **월간 리듬 유지** — 분기 업데이트만으로는 부족합니다. 월간 루틴 필수.

9. **피드백 수집** — 가끔 투자자에게 "업데이트 형식이 도움이 되나요?" 물어보세요.

10. **템플릿 진화** — 이 템플릿을 자신의 스타일과 회사 맥락에 맞게 조정하세요.
